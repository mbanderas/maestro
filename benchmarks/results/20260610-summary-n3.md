# Benchmark results — 2026-06-10, n=3 per cell

> **Validity note (added 2026-06-10):** these runs were measured with `verify.cjs` visible in the work dir (pre-hidden-oracle runner). Visible tests inflate agent pass rates 20-60% (arXiv:2602.10975), so cells here likely overstate success in BOTH modes and are not comparable to post-fix results. See benchmarks/README.md "Hidden oracle".

Full suite, 6 tasks x 2 modes x 3 runs = 36 task-runs, all sequential.
n=3 is still a small sample — medians are reported, no significance
claims are made, and run-to-run wall-time noise is visible (single-run
outliers of +50% occurred). Same model, same CLI build, same day for
every cell; rows are comparable within this table only.

- CLI: Claude Code (claude.exe, native build), model `sonnet`
- Isolation: per-run `CLAUDE_CONFIG_DIR` (no global config, hooks, or MCP in either cell)
- ON = repo `AGENTS.md` + `CLAUDE.md` copied into work dir; OFF = absent
- Per-run cap: `--max-budget-usd 1.0` (never hit)
- Raw data: `20260610-040902-claude-sonnet.json`, `20260610-041419-claude-sonnet.json`, `20260610-041955-claude-sonnet.json`

## Reading (n=3, still small)

- **Success parity 36/36.** Both cells solve all six tasks every run.
  Expected on xs/s tasks: Maestro's Decision Gate routes trivial work
  single-agent, so ON should never *lose* here — these cells measure
  overhead, not capability.
- **ON overhead is consistent**: +$0.016-0.048 median cost per task,
  +10-90k median cache-read tokens — the price of reading the doctrine
  files. Largest on t05 (refactor, +2 median turns, +$0.048) and t06
  (audit, +2 median turns, +$0.040), where the doctrine's verification
  and edit-safety rules visibly add turns.
- **No wall-time penalty on simple fixes** (t01-t03 medians within
  noise); measurable on t05/t06 (+30% and +17% median wall).
- Cells where Maestro must *earn* the overhead (multi-file t07/t08),
  plus Codex/Gemini rows: see later sections of the protocol; only
  measured numbers are published.
