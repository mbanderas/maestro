Why our default config is bracketed by 67.6–68.3

  • Default panel = GPT-5.5 + Gemini 3.1 Pro, judge+synth = Opus 4.8.
  • Bench has no exact "GPT-5.5 + Gemini 3.1 Pro" row; nearest are Opus+GPT-5.5 67.6 and the 3-model 68.3.
  • No fabricated row — the proven, transferable result is fusion > best solo + synthesis lift.

The synthesis lift is the load-bearing claim

  • Opus+Opus self-panel 65.5 vs solo Opus 58.8 = +6.7 with zero architecture diversity.
  • Means: even all-local same-model fan-out wins — lift survives if a panel CLI underperforms.
  • De-risks the weakest panel member (e.g. solo Gemini 3.1 Pro 45.4).

Cost lever

  • Budget panel 64.7% at ~50% cost — beats solo Opus/GPT-5.5.
  • Local CLIs already on flat-rate subscriptions → marginal API cost $0.
  • Concurrency + token-budget caps bound the parallel-web-loop blowup.

Adversarial audit surface — attack these to shrink the open risks

Each open risk stated as a falsifiable claim with the exact check that would refute it and the current mitigation. Point your next audit panel here.

P1 · Each panel CLI performs web_search / web_fetch headlessly
risk a panel member with no web silently breaks fan-out semantics (Fusion requires web per panel)
refute by run codex exec / gemini -p / claude -p headless on a "what changed today" prompt; check for fresh-source citations
mitigation M0 GO/NO-GO gate; CLIs without web gated webTools:false + documented divergence; M6 shared MCP shim
P2 · Judge emits valid Analysis JSON reliably from a CLI
risk CLI free-text ≠ API structured output; frequent parse failure collapses to degrade path, losing the analysis lift
refute by 50-run judge harness; measure valid-JSON rate; if < ~90% the degrade path is load-bearing, not exceptional
mitigation strict-parse + few-shot schema + retry-once; degrade is Fusion-faithful, not silent failure
P3 · Codex/Gemini auth survives headless spawn on Windows .cmd shims
risk interactive-only auth or %VAR% expansion in cmd.exe wrapper breaks non-interactive runs
refute by cold-shell headless spawn of each CLI; assert non-empty stdout, exit 0; test prompt with shell metacharacters
mitigation reuse spawn-cli.ts CVE-2024-27980 hardening; promptVia:'stdin' avoids arg-expansion
P4 · N parallel agentic web-loops stay within cost & latency budget
risk each panel CLI loops ≤ max_tool_calls of web fetches × N in parallel → cost/latency blowup
refute by wall-clock + token-est a 3-panel run; compare to budget cap; force a slow panel, confirm semaphore + timeout bound it
mitigation bounded semaphore (cap 4), pre-flight token-budget abort, per-CLI max_tool_calls / max_completion_tokens flag templates
P5 · Recursion is bounded to exactly one level
risk a spawned CLI re-invokes the fusion CLI → unbounded fan-out
refute by have a panel CLI shell out to the fusion bin; assert it returns fusion_invocation_capped
mitigation FUSION_DEPTH env injected into children; depth ≥ 1 → capped
P6 · Synthesizer grounds in analysis and never majority-votes
risk synth degenerates into "pick the answer most models gave" — the exact failure Fusion avoids
refute by adversarial input where the lone correct panel disagrees with two wrong ones; assert final follows the correct minority
mitigation synth prompt explicitly forbids majority-vote; judge preserves unique_insights / contradictions for grounding
